# 《暗线调查》整体重设计方案

## 1. 这次重设计要解决什么

当前版本的主要问题不是“功能不够多”，而是“玩家不知道自己在查什么、点完为什么有用、界面上哪里能点”。

这次整体方案重写，目标只围绕四件事：

1. 用 5 天时间讲清楚一条连续推进的调查故事。
2. 每天开头给玩家一句明确的“今日悬念”。
3. 每次行动结束后，用一句很短的话告诉玩家“你发现了什么”。
4. 让主界面只服务当前主线叙事，其他信息全部退到辅助层。

这次还要再明确一件事：

- 当前版本的首要目标不是做高度自由的调查模拟，而是做一款以线性叙事为核心的互动式文字游戏。

一句话定义这款游戏：

- 玩家扮演临时接案的调查员，在 5 天内沿着“审批干预 -> 平台代持 -> 实际控制”三条主线，逐步锁定幕后的 X 总。

## 2. 核心改造原则

### 2.1 线性叙事优先于分支设计

这款游戏当前阶段不再追求“多路线调查”和“多个同级行动入口”。

首版必须把重心放在：

- 剧情背景写清楚
- 每一天为什么开始写清楚
- 中间每一步为什么推进写清楚
- 每个场景和对话的文字质量打磨好

也就是说，玩家体验的核心不是“自由选择去哪查”，而是：

- 看剧情
- 做一次关键行动
- 在行动中完成轻答题或关键对话
- 得到新的剧情推进

### 2.2 玩家每次只需要知道三件事

主界面上，玩家任何时刻都只需要看懂这三件事：

1. 刚发现什么
2. 现在怀疑什么
3. 下一步去哪

除此之外的内容，例如证据清单、关系图、人物档案、历史记录，全部收进辅助抽屉或二级页面，不抢主叙事。

### 2.2 用“悬念”带玩家，而不是用“系统说明”带玩家

玩家每天进入时，不看“今日任务说明”，而是看到一句悬念式提示。

好的悬念应该满足：

- 一句话
- 不是术语堆砌
- 明确今天要查的核心问题
- 能自然引出今天的主行动

例如：

- “审批页是谁改的，为什么没人留痕？”
- “这些平台公司为什么总是刚好出现在关键位置？”
- “真正拍板的人，为什么从不在纸面上出现？”

### 2.3 行动结束后的反馈必须像“剧情推进”

每个行动结束后，不再给“线索 +1”“可信度 +5”这类系统反馈，而是给一句剧情句：

- “你确认审批意见被人改过，而且改动发生在会前一晚。”
- “你发现平台公司不是正常合作方，更像替别人站在台前。”
- “你第一次拿到口供，证明明面负责人并不做决定。”

这样玩家会感觉自己在破案，不是在点按钮结算。

### 2.4 主界面不再是信息堆场

主界面只负责：

- 显示今天的主线
- 显示最新发现
- 给出当天唯一的关键行动

主界面不负责完整展示：

- 全部证据
- 全部人物
- 全部关系
- 全部数值
- 全部历史日志

## 3. 新版内容结构

## 3.1 整体三幕结构

整个 5 天调查，按三幕推进：

### 第一幕：先确认这不是小问题

- 玩家接手一份异常项目材料
- 首要目标不是抓人，而是确认有没有人动过手脚
- 对应主线：`审批干预`

### 第二幕：查清谁在前面出面

- 玩家发现平台公司重复出现
- 开始怀疑它们不是正常合作方，而是在替别人挡在前面
- 对应主线：`平台代持`

### 第三幕：把真正的控制者逼出来

- 玩家已经知道这不是孤立问题
- 接下来要确认是谁在最后拍板、谁拿走了实际利益
- 对应主线：`实际控制`

## 3.2 5 天主线叙事脚本框架

### 第 1 天：先确认审批有没有被动过

- 今日悬念：
  - “审批页到底是正常修改，还是有人故意动过手脚？”
- 今日主线：
  - 玩家从档案材料入手，先确认异常不是普通手续疏漏
- 当日唯一行动：
  - 去 `A 银行档案室`
- 今日关键发现类型：
  - 审批页改动
  - 签批时间异常
  - 访客记录异常
- 行动结束反馈示例：
  - “你确认审批意见被改过，而且改动发生在正式会签前。”
- 当日收束句：
  - “现在可以确定，这不是一次普通流程失误。”

### 第 2 天：平台公司为什么总在关键位置

- 今日悬念：
  - “如果审批已经被动过，那平台公司为什么也刚好卡在关键节点上？”
- 今日主线：
  - 玩家开始追资金与合同路径，判断平台是否只是名义上的参与方
- 当日唯一行动：
  - 去 `数据核验中心`
- 今日关键发现类型：
  - 资金路径绕行
  - 平台重复出现
  - 合同位置异常
- 行动结束反馈示例：
  - “你发现项目资金先绕进平台，再回到项目链条里。”
- 当日收束句：
  - “平台已经不像普通合作方，更像被安排在前面的壳。”

### 第 3 天：明面上的人是不是只负责出面

- 今日悬念：
  - “这些平台负责人一直在说话，但他们真的是决定的人吗？”
- 今日主线：
  - 玩家开始接触项目经办人和外围联系人，验证“谁在出面、谁在决定”
- 当日唯一行动：
  - 去见 `项目经办人`
- 今日关键发现类型：
  - 口供松动
  - 聊天暗示
  - 名单不是经办人能改
- 行动结束反馈示例：
  - “你第一次拿到口供，证明明面负责人不是真正拍板的人。”
- 当日收束句：
  - “现在问题变成了：那真正点头的人是谁？”

### 第 4 天：真正拍板的人开始浮出来

- 今日悬念：
  - “如果前面的人都只是出面，那最后拍板的人为什么始终不留名字？”
- 今日主线：
  - 玩家追夜间接触、会议残页、隐性传话链，逼近核心人物
- 当日唯一行动：
  - 去 `夜间私下会面点`
- 今日关键发现类型：
  - 夜间碰面
  - 固定传话对象
  - 利益拆分台账
- 行动结束反馈示例：
  - “你发现每笔关键付款都要先给同一个人过目。”
- 当日收束句：
  - “整条线背后，已经出现了一个始终不露面的中心人物。”

### 第 5 天：把 X 总锁死

- 今日悬念：
  - “你已经知道谁在后面，但手里的东西够不够让他没法再否认？”
- 今日主线：
  - 玩家整理三条主线，补齐最后缺口，并与核心人物正面交锋
- 当日唯一行动：
  - 打开 `证据板`，补全三条命题
- 今日关键发现类型：
  - 控制备忘
  - 关键口供
  - 三线闭环
- 行动结束反馈示例：
  - “你把审批、平台和利益去向三条线连到了一起。”
- 当日收束句：
  - “现在已经不是怀疑，而是一条完整的控制链。”

## 3.3 每天固定叙事模板

为了让叙事稳定、易懂，每天都使用同一套结构：

1. 今日悬念
2. 昨天确认了什么
3. 现在怀疑什么
4. 今天唯一要做的行动
5. 行动结束后的新发现
6. 当日收束句

这样玩家不会迷路，也不会每次重新理解玩法。

## 4. 新版主界面方案

## 4.1 主界面只讲“当前主线”

主界面顶部固定一个 `当前主线` 区域，始终显示今天最重要的一句话。

内容只在两种状态中切换：

- 还没行动时：显示 `今日悬念`
- 做完行动后：显示 `最新发现`

例如：

- “今天要查清：平台公司为什么总在关键位置出现？”
- “你刚确认：项目资金先绕进平台，再回到项目链条里。”

这块区域是整页的叙事锚点，始终固定在顶部，不跟着页面内容滚走。

## 4.2 主界面信息架构

主界面只保留三个层次：

### A. 顶部固定区：当前主线

- 今日悬念或最新发现
- 一句话，不超过两行
- 永远固定在顶部

### B. 中部主叙事区：三件事

按顺序展示：

1. 刚发现什么
2. 现在怀疑什么
3. 下一步去哪

这三个信息直接写在页面主体上，用连续正文或短段落展示，不做信息卡，不做堆叠卡片。

### C. 行动区：当天唯一行动

- 只保留一个行动按钮
- 明确写出“去哪里”和“为了查什么”
- 玩家不再在主界面做行动分支选择
- 行动按钮下方可以配一句很短的提示，说明这一步的目的

### D. 辅助入口区：弱化到页面边缘

辅助信息全部收起来，只在需要时展开：

- 证据清单
- 线索记录
- 关系图
- 角色档案
- 案卷摘要

## 4.3 为什么这样改

这套结构解决了三个旧问题：

1. 玩家第一眼就知道当前故事在讲什么。
2. 玩家不会被一堆同权重信息淹没。
3. 玩家不会把正文叙事和行动按钮混在一起。

## 5. UI 设计方向

## 5.1 视觉基调

整体视觉改为浅色系，避免当前版本太像“深色控制台”。

建议方案：

- 页面背景：米白或极浅灰
- 卡片背景：白色或暖白
- 主强调色：深蓝
- 辅助线条：浅灰蓝
- 正文文字：深灰
- 次级文字：中灰

推荐色彩倾向：

- 背景：`米白`
- 主色：`深蓝`

原因：

- 深蓝更像调查、档案、记录、冷静判断
- 米白背景更适合长时间阅读文本
- 可以自然拉开“信息区”和“行动区”的差异

## 5.2 非行动区域不再使用信息卡

这是这次 UI 的硬要求。

非行动区域包括：

- 当前主线
- 最新发现
- 当前怀疑
- 剧情背景
- 当日收束

这些内容统一直接放在页面主体上，用排版层级区分，不再包成一张张卡片。

视觉要求：

- 以标题、短段落、留白来组织信息
- 不加卡片边框
- 不做悬浮阴影
- 不做按压效果
- 不做像按钮一样的轮廓

玩家一眼就知道：这些是叙事内容，不是交互对象。

### 行动按钮

行动按钮用于：

- 前往场景
- 约谈人物
- 整理证据
- 发起关键行动

视觉要求：

- 明显实色或描边
- 带箭头或方向图标
- hover 时有颜色变化和轻微位移
- 按压时有缩放或阴影变化
- 整页只保留一个主要行动按钮

玩家一眼就知道：这是这一段剧情里唯一需要点击推进的地方。

## 5.3 页面顶部固定“当前主线”

整个游戏最重要的 UI 改造，就是加一个固定顶部条。

固定条内容：

- 今天的核心悬念
  或
- 刚刚确认的最新发现

作用：

- 让玩家无论滚到哪里，都不会忘记今天到底在查什么
- 保证剧情推进感始终在线
- 代替传统的“任务栏”

这个区域要更像“案情标题条”，而不是游戏 HUD。

## 5.4 整体风格要像“读一份正在推进的案情”

界面气质要简洁、克制，不要堆满方块、标签、进度条和说明字。

建议方向：

- 大留白
- 短句
- 单列阅读优先
- 一个页面只强调一件事
- 行动按钮集中展示
- 辅助内容藏在抽屉里

不要做成：

- 多列仪表盘
- 满屏卡片矩阵
- 到处都是等权重按钮

## 6. 交互逻辑说明

## 6.1 一天的完整交互流程

每一天都按同一节奏运行：

1. 玩家进入当天
2. 看到一句今日悬念
3. 看到昨天确认了什么
4. 看到现在最怀疑什么
5. 点击当天唯一行动
6. 完成一个短行动
7. 收到一句最新发现
8. 主界面自动更新下一步建议
9. 当天结束时给一句当日收束

核心原则：

- 不让玩家自己在一堆入口里找剧情
- 而是由剧情推着玩家往前走

## 6.2 单次行动的结构

每次行动都拆成 4 步：

1. 进入场景或对话
2. 处理一个明确对象
3. 得到一句发现
4. 返回主界面

单次行动结束时，只给玩家一句结果，不展开大段系统说明。

句式模板：

- “你发现……”
- “你确认……”
- “这说明……”

例如：

- “你发现审批页的改动发生在正式签字前一晚。”
- “你确认平台账户先收了一笔不该出现的咨询款。”
- “这说明明面上的负责人并不控制最后决定。”

## 6.3 辅助信息如何收起

根据你的要求，证据、线索、关系图这些内容不应占据主界面中心。

因此建议：

- `证据`：放进右侧抽屉
- `线索记录`：放进底部抽屉
- `关系图`：做成二级页
- `角色档案`：做成二级页

主界面默认不展开这些内容。

只有玩家主动点开时才查看。

## 6.4 主线推进如何更明确

系统在后台继续保存完整证据与状态，但前台只翻译成玩家能懂的话。

例如后台判断：

- 玩家获得了某张审批证据
- 玩家已满足进入下一场景的前置条件

前台不写：

- “已解锁 scene_02”

而写：

- “你已经确认审批页有问题，下一步该去查钱是怎么走的。”

这就是整套交互的核心翻译原则。

## 7. 文案风格要求

这次方案的文案方向统一为：

- 短句
- 人话
- 少术语
- 先结果，后解释

硬性规则：

1. 一句话只说一层意思。
2. 优先说“发生了什么”，再说“这意味着什么”。
3. 少写抽象词，多写具体动作和结果。
4. 如果必须出现专业词，下一句立刻解释。

例如：

- 不写：
  - “你掌握了平台代持的初步迹象。”
- 改写：
  - “你发现平台公司并不独立，更像替别人站在前面。”

## 8. 首版建议保留的玩法深度

为了保证“好懂”而不是“空”，首版仍保留三种玩法层：

1. 场景中的轻答题
2. 关键人物对话
3. 章节之间的剧情推进

但每层都只做轻量版本：

- 场景内只处理 1 到 2 个对象
- 每天只做 1 次关键行动
- 对话每次只给 2 到 3 个选择

证据、关系和推理仍然保留，但它们在首版里更多承担“剧情支撑”作用，而不是让玩家自由组合路线。

这样既有互动性，又不会重新滑回“信息太多、分支太多、剧情反而变散”。

## 9. 设计结论

这一版整体方案的核心，不是把游戏做得更复杂，而是把复杂内容翻译成玩家能顺着走的叙事体验。

最终玩家感受到的应该是：

- 每天我都知道今天在查什么
- 每点一步我都知道自己刚发现了什么
- 主界面永远只告诉我最重要的那条线
- 其他信息随时可查，但不会先把我淹没
- 我不是在选路线，而是在顺着一条清晰剧情不断往前推进

如果后续按这份方案改代码，重点就不是“继续补组件”，而是：

1. 先重做主界面信息结构
2. 再重写 5 天叙事脚本
3. 再把证据与关系信息退到辅助层
4. 最后统一改成浅色叙事风格
