# 《暗流》—— 金融行业调查贪腐案件 网页文游 PRD

**版本**：v1.0\
**单局时长**：10分钟（10个决策回合）\
**目标平台**：移动端网页（竖屏，适配微信浏览器）\
**核心体验**：解谜 + 资源管理（在暴露风险与取证效率之间权衡）

***

## 一、背景设定（提取自真实案例，已脱敏改造）

**虚构机构**：华信银行（大型股份制商业银行）\
**核心人物**：已脱敏为"孙总"（原华信银行行长，2023年落马）\
**犯罪结构**（五层嵌套，对应五层谜题）：

> 2003年至2019年间，某银行高管利用信贷审批权，通过两名老部下作为代理人，设立两家投资平台公司。平台公司前台法人仅为"影子"，核心决策、人员工资、投资份额分配均由该高管实际控制。平台之下另有十多家项目公司，项目公司与行贿企业之间又各设空壳公司，形成多层嵌套的利益共同体。检方指控非法收受财物折合人民币9.795亿余元，一审死缓，终身监禁。

**游戏改编原则**：

- 不直接使用真实人名、机构名、地名
- 保留"信贷审批权→影子公司→代理人→投资平台→空壳公司→利益输送"的核心犯罪链条
- 将司法事实转化为可交互的谜题节点

***

## 二、核心循环（Core Loop）

| 元素       | 规则定义                                               |
| :------- | :------------------------------------------------- |
| **回合结构** | 游戏共10个"工作日"回合。每回合玩家执行**1个主动作** + 系统自动结算**1个反馈事件**。 |
| **可用动作** | ①调阅档案 ②约谈人员 ③追踪资金流 ④请示汇报 ⑤暂停观望（共5项，每回合5选1）         |
| **资源消耗** | 所有动作消耗1个工作日。约谈人员额外消耗"隐蔽点数"。                        |
| **进度推进** | 每回合结束后，时间压力-1。当时间压力=0时，强制进入结局判定。                   |

**输入→处理→输出**：

- **输入**：玩家选择动作A + 当前全局状态（证据完整度、警觉度、阻力、剩余天数、已激活线索标签）
- **处理**：系统调用该动作的收益公式与风险公式，计算属性变化，检定是否触发随机事件
- **输出**：更新后的全局状态 + 本回合事件结果 + 下回合可选动作列表（若某些动作因状态被锁定，则灰色不可选）

**设计意图**：10回合限制制造不可逆的紧迫感，单动作选择降低操作负担，适配移动端碎片化体验。

## 三、数值系统（Stat System）

### 3.1 全局变量（持久化，整局有效）

| 属性         | 初始值 | 上下界       | 获取/消耗途径                                                | 设计意图            |
| :--------- | :-- | :-------- | :----------------------------------------------------- | :-------------- |
| **证据完整度**  | 0   | \[0, 100] | 调阅档案+5\~15；追踪资金流+10\~25；约谈人员+0\~20（取决于话术对抗结果）；暂停观望+0   | 核心胜利条件，量化取证进度   |
| **嫌疑人警觉度** | 20  | \[0, 100] | 每回合基础+3；约谈人员+10\~20；调阅档案+2\~5；追踪资金流+5\~8；暂停观望-5；请示汇报-0 | 核心风险指标，模拟"打草惊蛇" |
| **机构阻力**   | 30  | \[0, 100] | 证据完整度每突破20/40/60/80阈值时+10；请示汇报-15；追踪资金流触及核心账户时+20      | 模拟内部保护伞反制       |
| **隐蔽点数**   | 3   | \[0, 5]   | 每回合自动回复1（上限5）；暂停观望额外+1；约谈人员消耗1                         | 限制高风险动作的频率      |
| **时间压力**   | 10  | \[0, 10]  | 每回合固定-1                                                | 硬性倒计时           |

### 3.2 关键线索标签（布尔数组，持久化）

| 标签ID              | 标签名称  | 激活条件                                                       | 作用                           |
| :---------------- | :---- | :--------------------------------------------------------- | :--------------------------- |
| `CREDIT_ABNORMAL` | 信贷异常  | 调阅档案检定大成功，或累计档案证据≥25                                       | 解锁"授信审批链"解密路径，降低后续追踪资金流难度10% |
| `SHADOW_COMPANY`  | 影子公司  | 追踪资金流检定大成功，或累计资金流证据≥30                                     | 解锁"公司关系图"解密子游戏，发现多层嵌套结构      |
| `AGENT_REVEAL`    | 代理人反转 | 约谈人员成功且隐蔽点数≥2，且`SHADOW_COMPANY`已激活                         | 发现两名"老部下"实为代理人，解锁关键证词        |
| `POWER_LOOP`      | 权力闭环  | `CREDIT_ABNORMAL` + `SHADOW_COMPANY` + `AGENT_REVEAL` 全部激活 | 解锁最终结局A的必要条件，拼接"权力变现"完整链条    |
| `INTERNAL_LEDGER` | 内部账本  | 机构阻力≥60后，通过特定事件触发                                          | 高风险高回报，直接+25证据完整度，但警觉度+15    |

**设计意图**：用布尔标签替代碎片化叙事，确保10分钟内玩家能清晰感知"我拿到了什么关键牌"。五层谜题对应五个标签，形成递进解锁结构。

## 四、分支与判定（Branching & Resolution）

### 4.1 通用检定机制

所有动作执行后，进行**隐蔽检定**：

```
检定结果 = D20 + 嫌疑人警觉度 × 0.2
难度等级 = 12（固定）
```

- 若结果 ≤ 难度：动作**成功**，获得全额收益
- 若结果 > 难度且 ≤ 难度+5：动作**部分成功**，获得50%收益，警觉度额外+3
- 若结果 > 难度+5：动作**暴露**，获得0收益，警觉度额外+10，触发"风声走漏"临时事件

### 4.2 专项动作规则

| 动作        | 输入条件   | 处理逻辑                                      | 输出结果                                                                                                    |
| :-------- | :----- | :---------------------------------------- | :------------------------------------------------------------------------------------------------------ |
| **调阅档案**  | 无特殊条件  | 证据完整度 + (D10+5)；警觉度 + (D4+1)；隐蔽检定         | 若`CREDIT_ABNORMAL`未激活且累计档案证据≥25，激活该标签。文本反馈引用"授信审批表、风控意见被修改痕迹、同一位高管反复批示的签字习惯"等背景元素                       |
| **约谈人员**  | 隐蔽点数≥1 | 消耗1隐蔽点数；进入话术对抗子系统；警觉度 + (D10+5)           | 成功则证据完整度 + (D10+10)；若大成功（D20=20）额外获得`AGENT_REVEAL`标签（需`SHADOW_COMPANY`前置）                               |
| **追踪资金流** | 无特殊条件  | 证据完整度 + (D12+8)；警觉度 + (D6+2)；机构阻力有30%概率+5 | 若`SHADOW_COMPANY`未激活且累计资金流证据≥30，激活该标签；若触及核心账户（D20≥18），机构阻力强制+20。文本反馈引用"咨询公司、投资平台、有限合伙企业、项目公司、空壳公司"等背景元素 |
| **请示汇报**  | 无特殊条件  | 机构阻力 - 15（最低降至0）；警觉度不变；证据完整度不变            | 若机构阻力≥60时执行，有50%概率触发"上级施压"事件                                                                            |
| **暂停观望**  | 无特殊条件  | 隐蔽点数+1；警觉度-5（最低0）；证据完整度+0                 | 无特殊标签关联                                                                                                 |

### 4.3 话术对抗子系统（约谈人员时触发）

```
玩家话术值 = D10 + 当前已激活线索标签数 × 3
NPC防御值 = D10 + 嫌疑人警觉度 × 0.15
```

- 话术值 > 防御值：约谈成功，证据完整度 + (D10+10)
- 话术值 ≤ 防御值：约谈失败，证据完整度 + 0，警觉度额外+5

**设计意图**：将"调查技巧"量化为已收集线索的乘数，鼓励玩家先低风险收集信息，再高风险突破，形成策略节奏。标签越多，话术越强，体现"证据链越完整，突破越有力"。

## 五、事件系统（Event System）

### 5.1 事件触发表

| 事件ID           | 事件名称  | 触发条件                       | 互斥关系            | 处理结果                                                                       |
| :------------- | :---- | :------------------------- | :-------------- | :------------------------------------------------------------------------- |
| `ANON_TIP`     | 匿名举报信 | 第3回合后，警觉度<<40，30%概率        | 与`UPPRESSURE`互斥 | 证据完整度+10，激活`CREDIT_ABNORMAL`或`SHADOW_COMPANY`中未激活的任意一项。文本引用"匿名举报信提及贷款审批异常" |
| `MEDIA_EXPOSE` | 媒体曝光  | 警觉度≥60，20%概率               | 无               | 警觉度+10，机构阻力-10（舆论压力降低内部阻力），但下回合"请示汇报"锁定。文本引用"财经媒体开始关注华信银行授信问题"             |
| `SUSPECT_MOVE` | 资产转移  | 警觉度≥70，每回合40%概率            | 无               | 嫌疑人开始转移资产，下回合"追踪资金流"收益-50%，但激活`INTERNAL_LEDGER`标签。文本引用"项目公司股权变更、空壳公司注销"    |
| `UPPRESSURE`   | 上级施压  | 机构阻力≥60且执行过"请示汇报"，50%概率    | 与`ANON_TIP`互斥   | 机构阻力+15，本回合收益-50%，连续2回合触发则强制进入结局D。文本引用"内部有人打招呼，要求'注意影响'"                   |
| `BREAKTHROUGH` | 重大突破  | 证据完整度≥50且警觉度<<60，第5回合后单次触发 | 整局仅一次           | 玩家下回合可同时执行2个动作（主动作+额外动作），隐蔽点数+1。文本引用"关键银行流水比对发现异常"                         |

### 5.2 状态变更写入规则

所有事件触发后，按以下优先级写入持久状态：

1. 布尔标签一旦激活，整局不可逆
2. 警觉度、机构阻力、证据完整度即时更新
3. 动作锁定状态（如"请示汇报"锁定）持续1回合后自动解除

**设计意图**：事件不是随机叙事装饰，而是对玩家前期策略选择的量化反馈，形成"早冒进→后期被动"或"早隐忍→后期爆发"的差异化路径。

## 六、进度与结局（Progression & Ending）

### 6.1 进度量化

- 无传统"章节"概念，以"证据完整度" + "已激活标签数"作为进度可视化
- 第5回合后，UI顶部增加"调查阶段"文字标识：
  - 证据<<20：初步排查（对应第一层谜题：信贷异常）
  - 20≤证据<<40：深入取证（对应第二层谜题：企业主关系网）
  - 40≤证据<<60：攻坚突破（对应第三层谜题：影子公司）
  - 60≤证据<<80：收网阶段（对应第四层谜题：代理人反转）
  - ≥80：闭环验证（对应第五层谜题：权力与利益闭环）

### 6.2 结局判定树（第10回合或强制触发时执行）

```
结局判定入口
├── 证据完整度 ≥ 80 ?
│   ├── 是 → 关键线索标签 ≥ 3（CREDIT_ABNORMAL + SHADOW_COMPANY + AGENT_REVEAL）?
│   │   ├── 是 → 结局A：铁证如山（权力闭环完整，移送司法）
│   │   └── 否 → 结局B：证据链缺损（有金额无闭环，定性困难）
│   └── 否 → 证据完整度 ≥ 60 ?
│       ├── 是 → 警觉度 = 100 ?
│       │   ├── 是 → 结局C：打草惊蛇（嫌疑人潜逃，调查中断）
│       │   └── 否 → 结局E：险胜收网（证据勉强足够，但链条有缺口）
│       └── 否 → 机构阻力 = 100 ?
│           ├── 是 → 结局D：调查叫停（上级干预，案件移交他处）
│           └── 否 → 结局B：证据不足（无法立案，不了了之）
```

### 6.3 结局条件量化表

| 结局           | 触发条件（布尔表达式）                                        | 文本摘要（引用背景元素）                                                   |
| :----------- | :------------------------------------------------- | :------------------------------------------------------------- |
| **结局A：铁证如山** | `证据≥80` AND `标签数≥3` AND `警觉<<100` AND `阻力<<100`    | 完整拼出"信贷审批→影子公司→代理人→投资平台→空壳公司→利益输送"五层闭环，涉案金额9.79亿，移送司法，主犯死缓终身监禁 |
| **结局B：证据不足** | `证据<<60` AND `阻力<<100` AND `警觉<<100`               | 发现异常但无法穿透多层嵌套结构，关键代理人未识别，案件移交纪检监察组另行调查                         |
| **结局C：打草惊蛇** | `警觉=100`                                           | 高频约谈导致嫌疑人察觉，两名代理人连夜出境，核心项目公司注销，资金链断裂无法追溯                       |
| **结局D：调查叫停** | `阻力=100`                                           | 内部保护伞发力，以"维护金融稳定"为由叫停调查，材料封存，调离原岗位                             |
| **结局E：险胜收网** | `60≤证据<<80` AND `警觉<<100` AND `阻力<<100` AND `时间=0` | 证据勉强足够立案，但多层嵌套结构未完全穿透，最终以"非国家工作人员受贿"降格处理                       |

### 6.4 多周目继承

- 若玩家单局激活标签数≥2，下周目初始隐蔽点数+1
- 若玩家曾达成结局A，解锁"专家模式"：初始警觉度40，时间压力8回合，但证据收益×1.2，且新增"审计追踪"动作（可替代"调阅档案"，收益+50%但警觉度+8）

**设计意图**：10分钟单局必须有多结局才有重玩价值，判定树用纯数值逻辑确保玩家能复盘"差哪一点"。

**交互规则**：

- 玩家拖拽节点可重新布局
- 点击节点显示该节点的已知信息（如"平台X：2015年成立，注册资本5000万，法人为张某，实际决策人为?"）
- 当`AGENT_REVEAL`激活后，两个平台公司的"实际决策人"节点从"?"变为"老部下A/老部下B"
- 当`POWER_LOOP`激活后，所有节点高亮并出现连接线，显示"信贷审批→投资入股→项目分红→空壳收益"的闭环

**设计意图**：将复杂的"多层嵌套公司结构"转化为可视化的解密游戏，降低理解门槛，同时提供"拼图完成"的成就感。

### 7.4 异常处理

- 若玩家网络延迟，所有按钮点击后进入1秒锁定态，防止重复提交
- 若玩家强退，本地缓存当前回合数与全局变量，下次进入询问"继续调查/重新开始"

### 9.3 关键算法伪代码

```typescript
// 执行动作
function executeAction(action: ActionType): RoundResult {
  const state = useGameStore();
  
  // 1. 计算基础收益
  let evidenceGain = 0;
  let alertnessGain = 0;
  let resistanceGain = 0;
  
  switch(action) {
    case 'archive':
      evidenceGain = rollD10() + 5;
      alertnessGain = rollD4() + 1;
      break;
    case 'interview':
      if (state.stealth < 1) return { error: '隐蔽点数不足' };
      state.stealth -= 1;
      evidenceGain = 0; // 由话术对抗子系统决定
      alertnessGain = rollD10() + 5;
      break;
    // ... 其他动作
  }
  
  // 2. 隐蔽检定
  const checkResult = rollD20() + state.alertness * 0.2;
  let resultType: 'success' | 'partial' | 'exposed';
  
  if (checkResult <= 12) {
    resultType = 'success';
  } else if (checkResult <= 17) {
    resultType = 'partial';
    evidenceGain *= 0.5;
    alertnessGain += 3;
  } else {
    resultType = 'exposed';
    evidenceGain = 0;
    alertnessGain += 10;
  }
  
  // 3. 更新状态
  state.evidence = Math.min(100, state.evidence + evidenceGain);
  state.alertness = Math.min(100, state.alertness + alertnessGain);
  state.resistance = Math.min(100, state.resistance + resistanceGain);
  
  // 4. 检查标签激活
  checkTagActivation();
  
  // 5. 触发随机事件
  const event = checkRandomEvent();
  
  // 6. 回合结束
  state.timePressure -= 1;
  state.currentRound += 1;
  
  // 7. 检查强制结局
  if (state.timePressure <= 0 || state.alertness >= 100 || state.resistance >= 100) {
    return { ...result, ending: calculateEnding() };
  }
  
  return { resultType, evidenceGain, alertnessGain, event };
}

// 结局计算
function calculateEnding(): EndingType {
  const state = useGameStore();
  const activeTags = countActiveTags();
  
  if (state.evidence >= 80 && activeTags >= 3 && state.alertness < 100 && state.resistance < 100) {
    return 'ENDING_A';
  }
  if (state.evidence < 60 && state.resistance < 100 && state.alertness < 100) {
    return 'ENDING_B';
  }
  if (state.alertness >= 100) return 'ENDING_C';
  if (state.resistance >= 100) return 'ENDING_D';
  if (state.evidence >= 60 && state.alertness < 100 && state.resistance < 100) {
    return 'ENDING_E';
  }
  return 'ENDING_B';
}
```
